# iTerminal 功能规划与优化建议

> 基于当前已实现功能的分析，整理缺失功能和操作优化建议。

---

## 当前功能概览

| 模块 | 已实现功能 |
|------|-----------|
| **连接管理** | SSH连接配置、分组管理、标签、状态检测、快速导入、连接测试、端口探测 |
| **终端** | xterm.js终端、多会话、搜索、复制粘贴、全屏、resize同步 |
| **文件管理** | SFTP浏览、上传下载、拖拽上传、新建/重命名/删除、chmod、远程压缩 |
| **系统监控** | CPU/内存/磁盘/基本信息、3秒自动刷新 |

---

## 一、核心缺失功能

### 1. 安全性（高优先级）

| 功能 | 说明 | 影响 |
|------|------|------|
| 密钥认证 | 代码中标记"暂未实现"，只支持密码认证 | 生产环境基本要求 |
| 密码加密存储 | 当前明文存 localStorage | 安全合规风险 |
| 主机密钥验证 | `check_server_key` 直接返回 true | 存在中间人攻击风险 |
| 会话超时锁定 | 无自动锁定机制 | 安全隐患 |

### 2. 连接管理（高优先级）

| 功能 | 说明 | 影响 |
|------|------|------|
| 连接导入/导出 | 无法批量导入导出配置 | 换机/备份麻烦 |
| 断线自动重连 | 网络波动后需手动重连 | 体验差 |
| 跳板机/代理 | 无 ProxyJump 或 SOCKS 代理支持 | 企业环境必需 |
| 连接历史 | 无最近连接记录 | 效率问题 |
| 批量操作 | 无法批量删除、分组移动 | 管理效率低 |

### 3. 终端功能（中优先级）

| 功能 | 说明 | 影响 |
|------|------|------|
| 快捷键系统 | 无快捷键配置（Ctrl+S保存、Ctrl+D关闭等） | 操作效率 |
| 命令片段/Snippets | 无常用命令保存和快速执行 | 运维效率 |
| 分屏终端 | 只能多标签，无法同屏分栏对比 | 对比操作不便 |
| 终端配置 | 无字体/字号/主题/编码自定义 | 个性化不足 |
| SFTP路径跳转 | 终端 `cd` 后文件管理器不同步 | 操作割裂 |

### 4. 文件管理（中优先级）

| 功能 | 说明 | 影响 |
|------|------|------|
| 文件预览 | 无文本/图片/代码预览 | 需下载查看 |
| 文件编辑 | 无在线编辑功能 | 修改文件麻烦 |
| 断点续传 | 大文件中断需重新上传 | 大文件传输风险 |
| 传输队列 | 无法排队、暂停、限速 | 批量传输管理弱 |
| 文件夹下载 | 代码提示"暂不支持" | 需先压缩再下载 |

### 5. 系统监控（中优先级）

| 功能 | 说明 | 影响 |
|------|------|------|
| 网络流量 | 无实时网速、连接数监控 | 运维盲区 |
| 进程管理 | 无法查看/杀死进程 | 需切换终端操作 |
| 服务状态 | 无 systemd 服务管理 | 服务管理不便 |
| Docker管理 | 无容器状态查看和操作 | 容器化环境必需 |
| 实时日志 | 无 tail -f 日志查看面板 | 日志查看不便 |

### 6. 用户体验（低优先级）

| 功能 | 说明 | 影响 |
|------|------|------|
| 主题切换 | 只有深色主题 | 个性化不足 |
| 多语言 | 仅中文 | 国际化受限 |
| 设置页面 | 无全局配置面板 | 配置分散 |
| 自动更新 | 无版本检查机制 | 升级不便 |

## 二、操作优化建议

### 1. 终端体验优化

#### 问题：工具栏占用空间，遮挡终端内容

**建议**：工具栏可设为自动隐藏，鼠标悬停或移到右上角时显示。

**实现思路**：
- 添加「自动隐藏工具栏」设置选项
- 默认状态改为小球形态
- 鼠标移到右上角区域时展开完整工具栏

---

#### 问题：右键菜单功能单一

**建议**：添加「在文件管理器中打开当前目录」「复制当前路径」等快捷操作。

**实现思路**：
- 后端获取终端当前工作目录（通过 `pwd` 命令或 shell 环境变量）
- 前端右键菜单增加相关选项
- 点击后打开文件管理面板并跳转到对应目录

---


### 2. 文件管理优化

#### 问题：文件列表是树形，不适合大量文件

**建议**：添加列表视图切换，支持按名称/大小/时间排序。

**实现思路**：
- 添加视图切换按钮（树形/列表）
- 列表视图支持表头排序
- 记住用户偏好的视图模式

---

#### 问题：上传无确认，直接覆盖同名文件

**建议**：添加文件冲突提示（覆盖/跳过/重命名）。

**实现思路**：
- 上传前检查目标路径是否存在同名文件
- 存在时弹出确认对话框
- 提供选项：覆盖、跳过、保留两者（自动重命名）

---

#### 问题：文件夹上传时无进度显示

**建议**：文件夹上传应显示总体进度。

**实现思路**：
- 上传前统计文件夹总大小和文件数
- 显示已完成文件数/总文件数
- 显示总体传输进度百分比

---

### 3. 连接管理优化

#### 问题：端口探测延迟 3 秒且阻塞启动

**建议**：改为后台异步检测，不延迟 UI 显示。

**实现思路**：
- 当前代码已异步，但启动延迟 3 秒可优化
- 检测完成后更新状态图标

---

#### 问题：快速导入格式固定

**建议**：支持解析 SSH URL 格式和常见格式。

**支持的格式示例**：
```
# SSH URL 格式
ssh://user@192.168.1.100:22
user@192.168.1.100

# 常见配置格式
名称: test
地址: 127.0.0.1
端口: 22
用户: test
密码: test

# 一行格式
test|192.168.1.100|22|root|password
```

---

### 4. 系统监控优化

#### 问题：每 3 秒刷新一次，无法调整

**建议**：添加刷新间隔配置，支持手动暂停。

**实现思路**：
- 监控面板顶部添加刷新间隔下拉框（1s/3s/5s/10s）
- 添加暂停/恢复按钮
- 暂停时显示最后更新时间

---

#### 问题：无告警阈值

**建议**：CPU/内存/磁盘超过阈值时变色或弹窗提醒。

**实现思路**：
- 设置页面添加告警阈值配置
- 超过阈值时进度条变红并闪烁
- 可选：系统通知弹窗提醒
- 添加告警记录历史

## 四、技术实现建议

### 密钥认证实现方案

```rust
// 后端使用 russh 的 key 认证
use russh::keys::*;

async fn authenticate_with_key(
    handle: &mut Handle,
    username: &str,
    key_path: &str,
) -> Result<bool, Error> {
    let key_pair = load_secret_key(key_path, None)?;
    handle.authenticate_publickey(username, Arc::new(key_pair)).await
}
```

### 断线重连实现方案

```typescript
// 前端监听断开事件，自动重连
const handleDisconnect = async (connectionId: string) => {
  const maxRetries = 3
  const retryDelay = 2000
  
  for (let i = 0; i < maxRetries; i++) {
    await sleep(retryDelay)
    try {
      await invoke('connect_ssh', { id: connectionId, connection })
      message.success('已自动重连')
      return
    } catch (e) {
      console.log(`重连失败 (${i + 1}/${maxRetries})`)
    }
  }
  message.error('重连失败，请手动连接')
}
```

### 命令片段数据结构
